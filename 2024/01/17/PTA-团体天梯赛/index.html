lower_bound函数

lower_bound 是一个在标准模板库（STL）中常用的函数，通常用于在一个已排序的范围内进行二分查找。它的目的是找到在范围内第一个不小于（即大于或等于）给定值的元素的位置。

下面是 lower_bound 函数内部逻辑的一个简化描述：

  1. 初始化指针或迭代器：函数开始时，设置两个指针或迭代器，分别指向要搜索的范围的起始和结束位置。

  2. 二分搜索：在搜索过程中，函数会不断将查找范围分为两半，选择其中一半继续搜索：

    • 计算中间位置：首先，找到当前查找范围的中间位置。
    • 比较中间元素与目标值：
      • 如果中间元素小于目标值，搜索继续在中间元素之后的部分进行，因为我们正在寻找的是第一个不小于目标值的元素。
      • 如果中间元素大于或等于目标值，搜索继续在中间元素以及之前的部分进行。

  3. 结束条件：当查找范围的开始位置超过或等于结束位置时，搜索结束。此时的开始位置就是第一个不小于目标值的元素的位置。如果所有元素都小于目标值，则该位置会是数组的末端。

  4. 返回结果：最终返回找到的位置的迭代器。

这种算法的时间复杂度是 ( O(\log n) )，其中 ( n ) 是搜索范围内的元素数量。由于它采用的是二分查找策略，因此在使用之前，必须确保范围内的元素是已排序的。

QA

Q

但是元素事实上并没有排序，可以使用lower_bound吗？

A

当使用 lower_bound 函数时，确实需要保证元素是按非降序排列的，也就是说，序列应该是排序的。在您的代码中，尽管原始输入数据可能未排序，但通过逐步构建的 top 向量实际上维护了一个有序序列。这个有序序列代表了目前为止找到的最长递增子序列的末端元素。

在这个过程中，每次当你使用 lower_bound 来查找 temp 的位置时，你其实是在查找在当前递增子序列中第一个不小于 temp 的位置。这允许你用 temp 替换找到的位置（如果有的话），从而维持这个递增序列的最小可能末端值。这个方法有效是因为它保留了递增序列的长度，同时使得后续有更大机会扩展这个序列。

因此，尽管原始输入可能未排序，你的算法通过维护 top 向量，保证了在每个时刻用于 lower_bound 搜索的序列都是有序的。这是最长递增子序列问题中常用的一种巧妙技术。

堆和栈

堆（Heap）和栈（Stack）是计算机内存中用于存储数据的两种重要区域，它们在数据存储和管理方面有各自的特点和用途。理解它们的区别对于编程尤其重要，因为它们影响了程序的性能和方式。

栈（Stack）

  1. 内存管理：栈是由操作系统自动管理的内存区域。当函数被调用时，其局部变量就会被分配在栈上。

  2. 大小限制：栈的大小通常有限。在大多数现代操作系统中，栈大小是固定的，一般为几百 KB 到几 MB。

  3. 访问速度：访问栈内存非常快，因为它具有连续的内存结构，且栈顶的地址是已知的。

  4. 生命周期：栈上的对象通常具有自动存储期。一旦创建函数的执行上下文（如函数调用），局部变量就会被创建；当上下文结束时（如函数返回），这些局部变量就会被销毁。

  5. 适用场景：适合存储函数参数、返回地址和局部变量。

堆（Heap）

  1. 内存管理：堆是由程序员手动管理的内存区域（在某些语言中，如Java和Python，由垃圾回收器管理）。当需要在程序的任何部分动态地分配内存时，会使用堆。

  2. 大小限制：堆的大小通常受到系统可用内存的限制，远大于栈。它可以用于分配大量的内存，如大数组和大对象。

  3. 访问速度：访问堆内存通常比访问栈内存慢，因为堆是在全局内存池中动态分配的。

  4. 生命周期：堆上的对象的生命周期不由它们的创建和销毁上下文决定。一旦分配，它们将持续存在，直到显式地被释放。

  5. 适用场景：适合存储需要在函数调用之间持续存在的数据，或者数据量太大无法在栈上分配的情况。

栈和堆的比较

  • 存储管理：栈是自动管理的，而堆需要手动管理（在某些高级语言中，如Java和Python，有垃圾回收机制）。
  • 生命周期：栈上的数据生命周期与函数调用相关，而堆上的数据生命周期更灵活。
  • 性能：栈上的数据访问更快，但空间有限；堆提供更多的空间，但性能开销更大。
  • 用途：栈适合于小量但快速的变量存储，堆适合于大量数据或需要跨函数持久存储的情况。

理解这些差异对于编写高效和稳定的程序至关重要。错误地使用堆和栈可能导致各种问题，如栈溢出、内存泄露或程序崩溃。

全局变量

在 C++ 中，全局变量不属于堆（Heap）也不属于栈（Stack）。全局变量以及静态变量（不管是全局的还是函数内的静态变量）被存储在一个单独的内存区域，通常称为“全局/静态存储区”或“数据段（Data Segment）”。

全局/静态存储区

  1. 生命周期：全局变量和静态变量的生命周期从程序开始执行时就开始了，直到程序结束时才销毁。

  2. 初始化：全局变量和静态变量在程序开始执行之前就已经被初始化。如果没有显式地初始化，它们会被自动初始化为零（对于基本数据类型）。

  3. 存储位置：这些变量不是存储在常规的栈或堆内存区域，而是在一个专门的内存区域，这有助于它们保持在整个程序执行过程中的持久性。

  4. 访问范围：全局变量可以在整个程序中任何位置被访问（除非它们被限制在某个特定的命名空间内）。

区别于堆和栈

  • 与栈的区别：栈用于存储局部变量和函数调用的上下文。栈上的变量通常在函数调用结束时被销毁。

  • 与堆的区别：堆用于动态内存分配，程序员可以控制何时分配和释放内存。堆上的变量在手动释放之前不会被销毁。

重要性

理解这些不同类型的存储区域对于编写高效和稳定的程序非常重要。错误地管理这些不同类型的内存可能导致内存泄漏、数据不一致或程序崩溃等问题。全局变量的使用应该谨慎进行，因为它们可以从程序的任何地方被访问和修改，这可能导致代码难以理解和维护。
