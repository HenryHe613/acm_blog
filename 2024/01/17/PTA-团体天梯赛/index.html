lower_bound函数

lower_bound 是一个在标准模板库（STL）中常用的函数，通常用于在一个已排序的范围内进行二分查找。它的目的是找到在范围内第一个不小于（即大于或等于）给定值的元素的位置。

下面是 lower_bound 函数内部逻辑的一个简化描述：

  1. 初始化指针或迭代器：函数开始时，设置两个指针或迭代器，分别指向要搜索的范围的起始和结束位置。

  2. 二分搜索：在搜索过程中，函数会不断将查找范围分为两半，选择其中一半继续搜索：

    • 计算中间位置：首先，找到当前查找范围的中间位置。
    • 比较中间元素与目标值：
      • 如果中间元素小于目标值，搜索继续在中间元素之后的部分进行，因为我们正在寻找的是第一个不小于目标值的元素。
      • 如果中间元素大于或等于目标值，搜索继续在中间元素以及之前的部分进行。

  3. 结束条件：当查找范围的开始位置超过或等于结束位置时，搜索结束。此时的开始位置就是第一个不小于目标值的元素的位置。如果所有元素都小于目标值，则该位置会是数组的末端。

  4. 返回结果：最终返回找到的位置的迭代器。

这种算法的时间复杂度是 ( O(n) )，其中 ( n ) 是搜索范围内的元素数量。由于它采用的是二分查找策略，因此在使用之前，必须确保范围内的元素是已排序的。

QA

Q

但是元素事实上并没有排序，可以使用lower_bound吗？

A

当使用 lower_bound 函数时，确实需要保证元素是按非降序排列的，也就是说，序列应该是排序的。在您的代码中，尽管原始输入数据可能未排序，但通过逐步构建的 top 向量实际上维护了一个有序序列。这个有序序列代表了目前为止找到的最长递增子序列的末端元素。

在这个过程中，每次当你使用 lower_bound 来查找 temp 的位置时，你其实是在查找在当前递增子序列中第一个不小于 temp 的位置。这允许你用 temp 替换找到的位置（如果有的话），从而维持这个递增序列的最小可能末端值。这个方法有效是因为它保留了递增序列的长度，同时使得后续有更大机会扩展这个序列。

因此，尽管原始输入可能未排序，你的算法通过维护 top 向量，保证了在每个时刻用于 lower_bound 搜索的序列都是有序的。这是最长递增子序列问题中常用的一种巧妙技术。
